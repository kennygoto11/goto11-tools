What Each Piece Does (Grade 5 Version)

Claude (Cowork)

Think of it as: Your assistant in the workshop

What it does: This is where you and I talk. When you say "change the headline" or "update the pricing," I make the edit right here.

Why it's here: So you never have to touch code. You just tell me what you want, and I do the technical work.

  • Edits files based on your instructions
  • Saves changes to GitHub (the filing cabinet)
  • Builds new tools and proposals

GitHub

Think of it as: A filing cabinet with a time machine

What it does: Stores every version of every file. If something breaks, we can always go back to the version that worked.

Why it's here: Two reasons — safety (nothing is ever lost) and automation (it tells Netlify "hey, something changed!").

  • Keeps a backup of everything
  • Tracks every change ever made
  • Sends a signal to Netlify when files update

Netlify

Think of it as: The delivery truck + the storefront

What it does: Hosts your tools on the internet so people can visit them. When GitHub says "new version!", Netlify grabs it and puts it live — automatically.

Why it's here: Without Netlify, your tools would just be files on a computer. Netlify makes them into live websites anyone can visit.

  • Turns files into live websites
  • Auto-updates when GitHub changes
  • Collects emails via the gate form
  • Sends you notifications of new visitors

The Magic: You Just Talk to Me

When everything is wired up, your workflow is: "Hey Claude, change X on the calculator" → I edit the file → it goes to GitHub → Netlify publishes it live. You never touch code, you never drag and drop, you never log into anything. The whole pipeline runs automatically from our conversation.
