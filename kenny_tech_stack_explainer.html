Claude — your operator

Think of Claude as a contractor you hire. You describe what you want, Claude builds it and hands you the finished product.

Claude writes the HTML, CSS, and JavaScript code that makes your tools work. It also connects to Notion, Slack, Gmail, Google Calendar, Netlify, and more through MCP connectors — so it can read your data and take actions on your behalf.

What Claude can't do: It can't directly push files to GitHub or Netlify. That's why you copy-paste code from Claude into GitHub, and GitHub auto-sends it to Netlify.

GitHub — your filing cabinet

Think of GitHub as a cloud-based filing cabinet where every version of every file is saved. If you mess something up, you can always go back to the previous version.

Your two GitHub repos right now:

goto11-tools — holds everything for app.goto11.ca (your internal hub)
goto11-client-tools — holds everything for tools.goto11.ca (client-facing tools)

When you edit a file on GitHub and click "Commit," that's like clicking Save. Netlify watches these repos and automatically re-publishes your site whenever you save.

Netlify — your web host

Think of Netlify as the building where your websites physically live. GitHub stores the blueprints; Netlify is the actual building people walk into.

Netlify takes your HTML files from GitHub and makes them accessible at real URLs. It also handles your custom domains (app.goto11.ca, tools.goto11.ca), SSL certificates (the padlock icon), password protection, and form submissions from your gate pages.

The magic: every time you save a file on GitHub, Netlify automatically rebuilds your site in about 15-30 seconds. You never have to manually "publish."

Replit — a code playground

Think of Replit as a workshop with tools already set up. You walk in, start building, and your creation is instantly live — no setup, no GitHub, no Netlify needed.

Replit is an all-in-one code editor and hosting platform in the browser. It's useful for quick prototypes or apps that need a backend (server-side code), which Netlify doesn't handle well. You haven't used it much yet, but it's an option for more complex tools down the road.

API — application programming interface

Think of an API as a restaurant menu. You don't walk into the kitchen — you read the menu (the API), place your order (send a request), and get your food back (receive a response). The kitchen does the work; you just interact through the menu.

When your LinkedIn Writer tool talks to Claude's AI to generate a post, it's using Claude's API. When Netlify checks GitHub for new files, it's using GitHub's API. Every tool-to-tool connection runs through APIs.

You don't need to build APIs yourself — you just use tools that already have them.

MCP — model context protocol

Think of MCP as power adapters for Claude. Each adapter lets Claude plug into a different tool — Notion, Slack, Gmail, Netlify, Google Calendar — and read or do things inside it.

MCP is what lets Claude create a Netlify site, search your Notion database, read your Slack messages, or check your calendar — all from inside this chat. Without MCP, Claude would just be a text generator with no ability to touch your real tools.

You manage which connectors are active in Claude's settings. Each one you turn on is like plugging in another adapter.
